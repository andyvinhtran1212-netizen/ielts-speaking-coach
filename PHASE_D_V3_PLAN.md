## 4. Data model plan
Bắt buộc áp dụng bài học B: RLS phải có đủ `USING` và `WITH CHECK`. Mỗi file migration phải kèm rollback DDL script ở cuối comment.

- **`021_vocabulary_exercises.sql`**
  - `id` (uuid, pk)
  - `target_vocab_id` (uuid, fk -> user_vocabulary, nullable - null nếu dùng seed)
  - `exercise_type` (varchar: 'D1' hoặc 'D3')
  - `content_payload` (jsonb - chứa prompt, câu gốc bị đục lỗ, đáp án...)
  - `status` (varchar: 'draft', 'published', 'rejected' - mặc định 'draft')
  - `created_at` (timestamptz)
  - **RLS:** Bắt buộc có policy riêng (Public read published + Admin write). Lưu ý kiểm tra auth mechanism của app (`users.is_admin` hay jwt claim) để config đúng:
    ```sql
    ALTER TABLE vocabulary_exercises ENABLE ROW LEVEL SECURITY;
    -- Public can SELECT published exercises
    CREATE POLICY "Public read published exercises"
      ON vocabulary_exercises FOR SELECT
      USING (status = 'published' OR auth.jwt()->>'role' = 'admin');
    -- Only admin can INSERT/UPDATE/DELETE
    CREATE POLICY "Admin write exercises"
      ON vocabulary_exercises FOR ALL
      USING (auth.jwt()->>'role' = 'admin')
      WITH CHECK (auth.jwt()->>'role' = 'admin');
    ```

- **`022_vocabulary_exercise_attempts.sql`**
  - `id` (uuid, pk)
  - `user_id` (uuid, fk -> auth.users)
  - `exercise_id` (uuid, fk -> vocabulary_exercises)
  - `user_answer` (text - từ điền vào hoặc transcript)
  - `is_correct` (boolean)
  - `score` (float)
  - `feedback` (jsonb - output từ Claude cho D3)
  - `attempted_at` (timestamptz)
  - **RLS:** `USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id)`
  - Index: `(user_id, date_trunc('day', attempted_at))` để query rate limit nhanh.

## 5. D1 Fill-blank implementation
- **Content gen workflow:** Gọi Gemini sinh câu ví dụ đục lỗ, dựa vào từ vựng (từ user bank hoặc seed list). Chạy script batch off-peak.
- **Admin review tool (BẮT BUỘC):** Gemini sinh bài tự động chuyển status='draft'. Admin duyệt, sửa nếu cần, rồi đổi thành 'published'. Không auto-publish.
- **Exercise format (JSON):** `{ "sentence": "The government must ___ the impact.", "answer": "mitigate", "distractors": ["aggravate", "ignore"] }`
- **Attempt grading:** String match (lowercase, trim). Khớp 100% -> Đúng. Không dùng AI để chấm D1.
- **Frontend UI:** Giao diện điền từ đơn giản, feedback ngay lập tức xanh/đỏ.

## 6. D3 Speak-with-target implementation
- **Target word selection:** Ưu tiên random từ trong `user_vocabulary` (status=learning). Nếu không đủ, trộn thêm seed list.
- **Whisper STT Integration:** Giao diện cho phép user bấm nút record, upload file, server chạy Whisper nhận transcript. **Rate Limit Gate phải đặt TRƯỚC Whisper call** để chống cost spike.
  - **Flow:** Auth check -> Feature flag -> Rate limit check (block 429 nếu hết) -> Audio upload -> Whisper STT -> Claude -> Save attempt.
- **Claude Prompt Design:** Fork từ `claude_grader.py`. 
  - **BẮT BUỘC:** Schema schema định dạng field `evidence_substring: str` là REQUIRED (không default empty).
  - Phân tích ngữ cảnh xem có đúng nét nghĩa không.
- **Guards:** Chống hallucination mạnh tay:
  - Verify `headword.lower() in evidence_substring.lower()` AND `evidence_substring in whisper_transcript`.
  - Nếu Claude không trả về `evidence_substring` -> reject toàn bộ response, retry 1 lần, nếu vẫn fail -> return error "AI grading failed, please try again".
- **Frontend UI:** Nút Record to, hiển thị transcript sau khi thu, highlight xanh lá từ mục tiêu. Có fallback báo đỏ nếu quên dùng từ.

## 7. D3 Fallback plan (Quyết định sau tuần 7)
Chạy checkpoint test 30 transcript adversarial tests. Action matrix:
1. **FP < 15%:** Đạt. Ship D3 bình thường theo luồng chuẩn.
2. **FP 15 - 25%:** Cận ranh giới. Ship D3 với label "Beta" và cảnh báo UX "AI might not catch your word perfectly due to mic quality".
3. **FP > 25%:** Rủi ro quá cao. Downgrade khỏi MVP, khóa D3 lại qua feature flag, chỉ ship D1. MVP vẫn thành công bằng D1.

*Bắt buộc chạy Prompt Injection Test trong checkpoint này.*

## 9. Rate limiting
Bảo vệ cost và chống spam API (bài học: không chỉ chặn ở frontend).
- **Backend enforcement:** Viết Python decorator `@rate_limit_exercise(type='D3', limit=3)` áp vào route. Query DB đếm số record `vocabulary_exercise_attempts` của user_id trong ngày (UTC).
- **Values:** 
  - D1: Không giới hạn (bấm chuỗi string, chi phí ~0).
  - D3 Free: 3 attempts/ngày (tránh abuse Whisper + Claude).
  - D3 Paid (future): 15 attempts/ngày.
- **Response:** Trả HTTP 429 Too Many Requests kèm message json rõ ràng.
- **Reset:** Nửa đêm UTC.

  - Test status transition: draft → published, draft → rejected.

- **Template Parity check**: Verify 3 HTML pages mới gọi đủ `initSupabase()` và CDN auth qua script cụ thể:

## 18. Risks + mitigations (Bài học: Triệu chứng giống ≠ Root cause giống)
- **D3 Prompt Injection:** Chặn ngay từ đầu bằng bộ test `test_d3_injection.py` và prompt grounding.
- **Rate Limit Bypass:** Thực thi logic đếm ở Database/Redis thay vì localStorage của Frontend.
- **Content quality D1 kém/rập khuôn:** Buộc qua Admin review gate, tuyệt đối không bypass Auto-publish.
- **Whisper cost spike:** Khóa chặt bằng D3 free (3 attempts) và block backend TRƯỚC khi gọi Whisper.
- **D3 quality không đạt:** Chấp nhận Fallback plan (hạ cấp beta hoặc tắt D3 hoàn toàn), bảo vệ MVP Phase D bằng D1.
- **Phase B regression:** Mọi thay đổi chung (services, models) phải chạy lại CI của Phase B.
- *Lưu ý debug:* Không gộp các UI bug (vd: lỗi ẩn nút D3) chung với logic bug (vd: API lỗi 403 RLS) dù triệu chứng chung là "user không chơi được D3".
